# Create a release

**Time:** ~15-20 min

**Purpose:** Learn how to create and manage releases in GitHub, which is an important part of the software development lifecycle.

**Context:** In professional software development, properly versioned releases help track changes, manage deployments, and communicate updates to users.

## Steps

### 1. Create an issue

Title: `[Task] Create a release`

### 2. Read documentation

1. Read [Managing releases in a repository](https://docs.github.com/en/repositories/releasing-projects-on-github/managing-releases-in-a-repository).
2. Skim [Semantic Versioning 2.0.0](https://semver.org/).

### 3. Create a tag

Create a tag `v0.1.0` for the current state of your fork.

### 4. Create a release

Create a release for `v0.1.0` with notes.

Use [Automatically generated release notes](https://docs.github.com/en/repositories/releasing-projects-on-github/automatically-generated-release-notes).

### 5. Document the release

Provide a link to the release in the issue description.

### 6. Close the issue

Close the issue once the release is created.

## Acceptance criteria

- [ ] Issue created
- [ ] Tag `v0.1.0` created
- [ ] Release created with automatically generated notes
- [ ] Link to release added to the issue description
- [ ] Issue closed
